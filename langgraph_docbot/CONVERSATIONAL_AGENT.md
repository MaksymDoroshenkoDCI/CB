# 💬 Conversational Requirements Agent

## Огляд

Додано **Conversational Requirements Agent** - агент, який веде діалог з користувачем для збору вимог перед генерацією IT-документації.

## ✨ Нові можливості

### 1. Multi-turn Dialog
- Агент задає структуровані питання користувачу
- Зберігає відповіді в пам'ять
- Адаптує питання на основі попередніх відповідей

### 2. Два режими роботи
- **Прямий режим**: Генерація документації з одного запиту (як раніше)
- **Режим інтерв'ю**: Діалог для збору вимог перед генерацією

### 3. API Endpoints
- `POST /conversation/start` - Почати новий діалог
- `POST /conversation/continue` - Продовжити діалог з відповіддю
- `POST /conversation/generate` - Згенерувати документацію після діалогу

### 4. Streamlit UI
- Новий інтерфейс з вибором режиму
- Інтерактивний діалог в режимі інтерв'ю
- Відображення історії діалогу

## 🚀 Використання

### Через Streamlit UI

1. Запустіть Streamlit:
   ```bash
   streamlit run ui/streamlit_app.py
   ```

2. Оберіть режим "Інтерв'ю (діалог)" в бічному меню

3. Натисніть "Почати інтерв'ю"

4. Відповідайте на питання

5. Після завершення діалогу натисніть "Згенерувати документацію"

### Через API

```python
import requests

# 1. Почати діалог
response = requests.post("http://localhost:8000/conversation/start", json={
    "project_name": "fintom8"
})
session_id = response.json()["session_id"]
question = response.json()["question"]

# 2. Продовжити діалог
response = requests.post("http://localhost:8000/conversation/continue", json={
    "session_id": session_id,
    "answer": "Система призначена для автоматизації продажів..."
})

# 3. Повторювати крок 2 до завершення діалогу

# 4. Згенерувати документацію
response = requests.post("http://localhost:8000/conversation/generate", json={
    "session_id": session_id
})
documentation = response.json()["documentation"]
```

## 🎯 Питання для збору вимог

За замовчуванням агент задає наступні питання:

1. Опиши основну мету системи. Що вона має робити?
2. Хто основні користувачі системи? Для кого вона призначена?
3. Які ключові функції повинні бути реалізовані? Опиши основні можливості.
4. Які технології плануються? (мови програмування, фреймворки, бази даних)
5. Які інтеграції потрібні? (зовнішні API, сервіси, системи)
6. Які вимоги до безпеки? (автентифікація, авторизація, захист даних)
7. Опиши архітектуру системи. Які основні компоненти?
8. Які є вимоги до розгортання та інфраструктури?

## 🔧 Налаштування

Питання можна налаштувати в `app/conversation_agent.py`:

```python
DEFAULT_QUESTIONS = [
    "Ваше питання 1",
    "Ваше питання 2",
    # ...
]
```

## 📝 Примітки

- Сесії зберігаються в `memory/conversation_sessions.json`
- Користувач може завершити діалог в будь-який момент, написавши "Готово" або "Згенеруй документацію"
- Агент може генерувати адаптивні питання на основі попередніх відповідей (через LLM)
