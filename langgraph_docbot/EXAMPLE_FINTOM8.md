# Приклад використання для проекту fintom8

## Як згенерувати документацію для fintom8

### Через Streamlit UI:

1. Запустіть Streamlit:
   ```bash
   streamlit run ui/streamlit_app.py
   ```

2. У полі "Назва проекту" введіть: `fintom8`

3. У полі опису введіть детальний опис системи, наприклад:

```
fintom8 - фінансова платформа для управління особистими фінансами та інвестиціями.

Технологічний стек:
- Backend: Python 3.11, FastAPI, SQLAlchemy
- Frontend: React 18, TypeScript, Material-UI
- База даних: PostgreSQL 15
- Кешування: Redis
- Аутентифікація: JWT токени
- Деплой: Docker, AWS ECS
- CI/CD: GitHub Actions

Основні модулі:
1. User Management - управління користувачами та профілями
2. Financial Accounts - підключення банківських рахунків через Open Banking API
3. Transactions - обробка та категорізація транзакцій
4. Budget Planning - планування бюджету та відстеження витрат
5. Investment Portfolio - управління інвестиційним портфелем
6. Analytics & Reports - аналітика та звіти
7. Notifications - сповіщення про важливі події

API інтеграції:
- Open Banking API для отримання даних з банків
- Stock Market API для отримання котирувань
- Email сервіс для сповіщень

Архітектура:
- Мікросервісна архітектура
- RESTful API
- Event-driven communication через message queue
- Моніторинг через Prometheus та Grafana
```

4. Натисніть "Згенерувати документацію"

### Через API:

```bash
curl -X POST "http://localhost:8000/generate" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "fintom8 - фінансова платформа...",
    "project_name": "fintom8"
  }'
```

## Що буде згенеровано:

Система створить повну технічну документацію, яка включає:

1. **Вступ та загальний опис** - що таке fintom8, основні цілі
2. **Архітектура системи** - компоненти, діаграми, взаємодія
3. **Технічний стек** - всі використані технології
4. **Функціональність** - детальний опис кожного модуля
5. **API та інтеграції** - endpoints, зовнішні сервіси
6. **База даних** - схема, моделі, зв'язки
7. **Безпека** - автентифікація, авторизація, захист
8. **Розгортання** - процес деплою, інфраструктура
9. **Тестування** - стратегія тестування
10. **Моніторинг** - логування, метрики
11. **Документація для розробників** - як почати роботу

Документ буде збережено у папці `generated_docs/` з назвою типу `doc_fintom8_YYYYMMDD_HHMMSS.txt`
